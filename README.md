# Astrodopamine

ДЕМО: [astrodopomine.ru](https://astrodopomine.ru)

## Первый запуск

В териминале VSCode:

```bash
npm install
npm run dev
```

Доступ сайта после запуска по ссылке [http://localhost:3000](http://localhost:3000). Изменения в файлах автоматически обновляют страницу, удобно.

## Команды

| Команда | Описание                                       |
|---------|------------------------------------------------|
| `npm run dev` | Запуск dev-сервера с live-reload               |
| `npm run build` | Продакшн-сборка (минификация CSS/JS) в `dist/` |
| `npm run backend` | Сборка без минификации (удобнее для backend)   |

**Важно:** в GitHub Pages сейчас уходит корневой путь проекта, нужно поменять folder на (`/dist`) чтобы все изменения подтянулись на домен.

## Структура проекта

```
src/
├── html/
│   ├── pages/            — страницы (index.html) — точки входа
│   └── includes/         — переиспользуемые куски HTML
│       ├── head.html     — <head> с мета-тегами и подключениями
│       ├── header.html   — шапка сайта
│       ├── footer.html   — подвал
│       └── sections/     — секции страницы (hero, faq, prices и др.)
├── scss/
│   ├── main.scss         — точка входа, импортирует всё остальное
│   ├── base/             — переменные, миксины, reset, базовые стили
│   └── components/       — стили по секциям (_hero.scss, _prices.scss и др.)
├── js/
│   ├── main.js           — главный скрипт
│   └── components/       — отдельные JS-модули (burger.js, footer.js)
└── assets/
    ├── images/           — изображения
    └── fonts/            — шрифты
```

## Зачем такая структура

Компонентный подход вместо «всё в одном файле»:

- **Навигация** — сразу видно где искать стили hero-секции или текст FAQ
- **Параллельная работа** — два человека не конфликтуют в одном файле
- **Поддержка** — изменение одной секции не ломает остальные
- **Переиспользование** — header/footer подключаются через `@@include`, не копируются

В общем такая структура больше подходит для дальнейшего масштабирования и работы в команде. Лучше это сделать в самом начале, чем потом страдать.
